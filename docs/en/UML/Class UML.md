### <span style="color:grey"> Class diagram at UML:</span>
The inheritance hierarchy can be represented in the form of a diagram.Unified modeling language (UML) is the standard established in this area.UML defines several types of diagrams that describe both the structure and software behavior.

A good tool to describe the inheritance hierarchy of functions blocks is the class diagram.

UML diagrams can be created directly on Twincat 3. Changes in the UML diagram have a direct effect on Pou.Therefore, functions blocks can be modified and modified through the UML diagram.

Each box represents a function block and is always divided into three horizontal sections.The upper section shows the name of the functions block, the central section lists its properties and the lower section lists all its methods.In this example, the arrows show the direction of the inheritance and always point to the main functions block.
